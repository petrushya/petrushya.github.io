Универсальный <DIV>

26 Окт, 2023


При создании этого простенького сайтика изначально использовалась стилизация HTML–элемента <body>, в частности применялись такие свойства как width и margin, для получения желаемого результата.

Всё выглядело миленько до использования браузерного расширения TWP. При выделении текста, значок перевода улетел за пределы страницы, там же отображался и перевод. После просмотра кода страницы, выяснилось, что элемент <div> TWP внедряется в <body> элемент перед закрывающим тэгом и использует style=”all: initial;”. Попытка игры со свойством padding корневого элемента <html> также ничего не изменила.

Пришлось обратиться опять же к MDN Web Docs, где сказано: On inherited properties, the initial value may be unexpected. После недолгих изысканий внимание привлёк тот же <div>, так как он характеризуется на MDN Web Docs как: As a “pure” container, the <div> element does not inherently represent anything. Instead, it’s used to group content so it can be easily styled using the class or id attributes, marking a section of a document as being written in a different language (using the lang attribute), and so on.

Может быть это и не совсем правильно, но после оборачивания всего кода, содержащегося в <body> элементе, дополнительным <div> элементом и переноса стиля из <body> в <div> элемент вид страниц сохранился, а TWP стал работать нормально. Вот и получается, что HTML–элемент <div> сохраняет свою актуальность, особенно при нестандартных ситуациях.
